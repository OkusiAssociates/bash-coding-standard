Part VIII — Conditional Expressions and Arithmetic

Bash has two test contexts: [[ ]] for file/string/regex and (( )) for arithmetic. The legacy [ ] POSIX test exists but is not used in modern Bash. This Part documents both contexts, their operators, and their precedence rules.

Chapters

  1. 8.1 [[ ]] overview — The modern conditional command.
  2. 8.2 File test operators — Single-operand tests on files.
  3. 8.3 File comparison operators — Two-operand file tests.
  4. 8.4 String operators — String comparison and inspection.
  5. 8.5 Pattern matching with == — Right-hand side of == (or =) inside [[ ]] is a glob pattern unless quoted.
  6. 8.6 Regex matching with =~ — Right-hand side of =~ is an ERE (extended regular expression).
  7. 8.7 Logical operators and grouping — Inside [[ ]], logical operators combine sub-expressions.
  8. 8.8 Quoting rules inside [[ ]] — [[ ]] is a reserved word, parsed specially.
  9. 8.9 Arithmetic context (( )) — (( expression )) evaluates expression as arithmetic, returns 0 if non-zero, 1 if zero.
  10. 8.10 Arithmetic operators and precedence — Bash arithmetic supports a rich operator set with C-like precedence.
  11. 8.11 Integer types, overflow, base prefixes — Bash arithmetic uses signed C intmax_t — typically 64-bit on Linux.
  12. 8.12 Floating-point — workarounds — Bash has no native floats.
  13. 8.13 let builtin — let evaluates its arguments as arithmetic expressions, returning failure if the last evaluates to zero.
  14. 8.14 The deprecated [ ] and test — POSIX test builtin and its [ ] synonym.
